# 实验指导：基于 Bi-LSTM 的中文情感分析

## 一、实验说明

### 1.1 本资料包随附文件

以下文件由发版方提供，**可直接使用或在其框架内补全**（以课程发行为准）：

| 文件 | 说明 |
|------|------|
| **`prepare_data.py`** | 从网络下载 `waimai_10k` 数据集、划分训练/测试集、用 jieba 构建 `vocab.json`。**一般无需修改**。 |
| **`train.py`** | 已包含：`WaimaiDataset`、DataLoader、训练/验证循环、`CrossEntropyLoss`、按测试集准确率保存 `model_best.pt`。**其中 `BiLSTMClassifier` 的循环部分已挖空**，须由你补全。 |
| **`predict.py`** | 从 `train.py` **导入** `BiLSTMClassifier` 与 `get_device`，完成加载检查点、单句编码与预测。**不要在本文件内再写一份模型类**，以免与 `state_dict` 不一致。 |
| **`requirements.txt`** | Python 依赖列表（含 PyTorch 等），建议用其安装环境。 |

### 1.2 必须由你完成的代码（核心）

| 位置 | 内容 |
|------|------|
| **`train.py` 中类 `BiLSTMClassifier`** | 在 **`__init__`** 中定义双向 LSTM 各层、各向所需的**可学习参数**；在 **`forward`** 中实现从词向量到句向量、再到分类 logits 的**完整前向计算**。 |
| **实现约束** | 为强化对 **LSTM 门控与结构**的理解，**禁止**使用 `nn.LSTM`、`nn.LSTMCell`、`nn.RNN`、`nn.RNNCell`、`nn.GRU`、`nn.GRUCell` 等封装的循环层。允许使用 `nn.Linear`、`nn.Embedding`、`nn.Parameter`、`nn.Dropout` 及 `torch` 张量运算等，**自行按教材公式**实现 LSTM 与双向、多层时序展开（及合理的 padding 处理）。 |
| **与实验指导 3.3 的对应** | 须实现与下文「数据流」一致：最后一层、最后时刻的前向/后向隐状态 **concat** 后接 `Dropout` 与全连接。类中已保留 `self.embedding`、`self.dropout` 与 `self.fc` 的衔接关系，**不得删除**其设计意图，只在中间补全你的 Bi-LSTM 计算。 |

**不要求**你从零重写 `prepare_data.py` 的数据下载逻辑、也不要求重写 `train.py` 中的 Dataset/DataLoader/训练主循环；**要求**你亲手实现 **Bi-LSTM 主体**，并能训练出模型、用 `predict.py` 做推理。

## 二、环境与数据准备

### 2.1 依赖环境

建议使用 **Python 3.8** 及以上版本。在项目目录下可执行：

```bash
pip install -r requirements.txt
```

若未提供 `requirements.txt`，至少需安装：

```bash
pip install torch pandas jieba scikit-learn datasets
```

### 2.2 数据与词表生成

在脚本所在目录执行（需联网，首次运行会从 HuggingFace 下载 `XiangPan/waimai_10k` 数据集）：

```bash
python prepare_data.py
```

执行成功后，将在当前目录生成如下文件：

| 文件 | 说明 |
|------|------|
| `train.csv` | 训练集，UTF-8 编码，包含 `label` 与 `text` 两列 |
| `test.csv` | 测试集，格式同训练集 |
| `vocab.json` | 词表文件：`"<PAD>"` → 0，`"<UNK>"` → 1，其余词从 2 开始编号 |

### 2.3 数据处理约定

**数据划分：**

- 训练集与测试集按 **8:2** 划分；
- 随机种子：`random_state=42`；
- 采样方式：**分层抽样**（`stratify=label`）。

**词表构建规则：**

- 基于**训练集**文本；
- 使用 **jieba** 分词；
- 仅保留词频 **≥ 2** 的词。

### 2.4 数据字段说明

| 字段 | 说明 |
|------|------|
| **`label`** | `0`：负向情感；`1`：正向情感 |
| **`text`** | 原始中文评价文本；`train.py` 中 `WaimaiDataset` 已按 jieba 分词与 `vocab.json` 映射，你须保证**推理时**（如 `predict.py`）**与训练时**分词、截断、填充规则一致。 |

## 三、实验任务与代码分工

发版方提供的 `train.py` 中，**以下部分已写好**（在理解基础上可直接使用，一般不必推倒重来）：

- 随机种子、`load_vocab`（若你单独需读词表可复用）；
- **`WaimaiDataset`**：jieba 分词、id 序列、**`max_len` 截断与 `<PAD>` 填充**；
- 训练/测试 **DataLoader**、**`train_one_epoch`**、**`evaluate`**（含 Accuracy）；
- **`main`**：命令行超参数、每轮在测试集上评估、**保存更优的 `model_best.pt`**（内含 `state_dict` 与 `hparams`）。

**须你补全的唯二核心**（均在 **`BiLSTMClassifier` 内**）：

1. **`__init__`**：除已给出的 `self.embedding` 与 `self.dropout`、`self.fc` 外，为「手写 Bi-LSTM」声明所需 **`nn.Linear` / `nn.Parameter` 等**（具体形状与层数与 `num_layers`、`hidden_dim` 等超参一致）。
2. **`forward`**：实现 **Embedding → 双向、多层时序前向（你自己写的 LSTM）→ 取最后一层最后时刻的前向/后向隐状态并 concat → dropout → 全连接**，输出 `(batch, 2)` 的 logits。须**删除/替换**类内占位用的 **`NotImplementedError`**，使训练能真正跑通。

`predict.py` 中通过 **`from train import BiLSTMClassifier, get_device`** 复用你的模型类；你完成 `forward` 并成功保存权重后，执行：

```bash
python predict.py --text "你的一句测试评论"
```

或省略 `--text` 从标准输入读入一行。若加载失败，请检查**是否只在一处**定义了 `BiLSTMClassifier`（在 `train.py`）。

### 3.1 与「手写 Dataset」等表述的关系

若课程早先要求「自行实现 Dataset」，发版方已在 `train.py` 中给出 `WaimaiDataset` 作为**标准参考**；**本实验的侧重**是你在 **`BiLSTMClassifier`** 中体现对 **Bi-LSTM 结构** 的掌握。若希望用自己的 Dataset 组织方式，需保证与**词表、长度、批形状**等仍与当前训练流程兼容（不建议无故改动发版中 DataLoader 的接口约定，除非你能承担兼容性后果）。

### 3.2 模型结构要求（与随附参考代码的衔接）

模型需从架构上体现：

- **Embedding** 层（发版中已作为 `self.embedding` 给出，一般保留并调用）；
- **双向、多层** LSTM 逻辑（**由你**用**基础层**实现，**不得**用 `nn.LSTM` 等黑盒层）；
- **Dropout** 与**全连接**到二分类（发版中 `self.dropout` 与 `self.fc` 已给出，**输入维为** `2 * hidden_dim`）。

使用 **`CrossEntropyLoss`** 进行训练（已在 `train.py` 主流程中接好）。

### 3.3 特征提取方式（重要）

数据流需与下述一致（你在 `forward` 中的张量操作应能实现该流程）：

```text
(batch, seq_len)
  → Embedding
  → Bi-LSTM（自行实现，非 nn.LSTM）
  → 取最后一层前向 + 后向在「整句有效结束位置」的隐状态
  → 拼接（concat）
  → (batch, 2 × hidden_dim)
  → Dropout
  → Linear
  → (batch, 2)
```

**说明**：对 padding 的句子，「最后一步」应对**非 PAD 的最后一个时间步**取隐状态；若发版中简化为对固定下标取状态，**建议在报告中说明**；更完善的掩码由你加分。

### 3.4 训练与评估

- 在训练集上训练（`train.py` 已提供循环）；
- 在 **`test.csv`** 上报告 **Accuracy**（`train.py` 中已打印每轮 `test_loss` 与 `test_acc`）；
- 更优检查点**自动保存**为 `model_best.pt`（与 `main` 中 `--save` 一致）。

### 3.5 预测模块

`predict.py` 已提供：加载 `model_best.pt`、**与训练时一致**的文本编码、输出类别与**两类概率**。**你**只需保证 `train.BiLSTMClassifier` 与保存的 `state_dict` 匹配，无需在 `predict.py` 再实现网络。
